要将 `dev01` 分支中不同于 `master` 的部分提取出来并创建一个新分支，我们需要先明确您希望如何处理这些差异的提交历史。

在 Git 中，通常有两种最常用的策略来实现这一目标：

1. **拣选（Cherry-pick）**：适用于您希望原封不动地保留 `dev01` 中某些特定提交（Commit）的历史记录。
1. **压缩合并（Squash Merge）**：适用于您只想要代码的最终差异，不在乎繁琐的历史记录，并希望将所有改动打包成一个全新的提交。

### 方法一：使用 Cherry-pick（保留提交历史）

如果您希望将 `dev01` 上的特定提交逐一复制到新分支上，请按照以下步骤操作：

- **步骤 1：查看差异提交** 首先，您需要找出 `dev01` 相对于 `master` 有哪些不同的提交。在终端中输入： `git log master..dev01 --oneline` 这会列出所有在 `dev01` 但不在 `master` 中的提交。请记下您需要提取的提交的哈希值（例如 a1b2c3d）。
- **步骤 2：创建并切换到新分支** 接下来，基于最新的 `master` 分支创建一个新分支（假设命名为 new-branch）并切换到该分支： `git checkout -b new-branch master`
- **步骤 3：拣选提交** 将您在步骤 1 中记下的提交应用到新分支上。如果您只需要几个特定的提交，可以用空格隔开： `git cherry-pick <提交哈希值1> <提交哈希值2>` 如果您想提取一段连续的提交范围（从较旧的提交到较新的提交），可以使用： `git cherry-pick <较旧提交哈希>^..<较新提交哈希>`
