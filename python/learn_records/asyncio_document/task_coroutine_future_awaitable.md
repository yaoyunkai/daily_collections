# Task, Coroutine, Future & Awaitable

> 通俗比喻（餐厅点餐模型）

- **Coroutine（协程） = 【菜谱】**。菜谱上写满了做菜的步骤（代码逻辑），但菜谱本身不能吃。如果你只拿着菜谱，什么都不会发生。
- **Future（未来对象） = 【取餐叫号器 / 小票】**。你点完餐后，前台给你的一个空盒子或小票。它代表一个“未来的承诺”：现在里面没东西（Pending），但等菜做好了，里面就会装满食物（Done/Result）。
- **Task（任务） = 【后厨流水线工人】**。工人拿着你的【菜谱】（Coroutine）去干活。同时，这个工人本身也自带了一个【取餐叫号器】（因为 Task 继承自 Future），当他按菜谱把菜做完时，叫号器就会响，通知你可以来拿菜了。

> 技术原理解析与逻辑推理

我们逐步拆解这三者的技术定义，并推理它们是如何结合在一起的：

### Coroutine (协程)：静态的逻辑蓝图

- **定义**：使用 `async def` 定义的函数。当你调用这个函数时，它**不会立即执行**，而是返回一个“协程对象”（Coroutine Object）。
- **特性**：它是惰性的（Lazy）。一个单纯的协程对象如果不被事件循环调度，或者不被 `await`，它就永远是一段死代码，毫无作用。

### Future (未来对象)：底层的状态容器

- **定义**：`asyncio.Future` 是一个底层的可等待对象（Awaitable）。它代表一个异步操作的**最终结果**。
- **特性**：它本质上是一个状态机，包含三种状态：`PENDING`（进行中）、`DONE`（已完成）、`CANCELLED`（已取消）。
- **作用**：它通常用于连接底层的回调式代码（Callback）和高层的 `async/await` 代码。当底层 I/O 准备就绪时，底层代码会调用 `future.set_result(value)`，这会唤醒正在 `await future` 的上层代码。

### Task (任务)：驱动协程运行的引擎

- **定义**：`asyncio.Task` 是 `Future` 的**子类**（`class Task(Future): ...`）。
- **核心作用**：它的主要职责是**包装一个 Coroutine**，并将其**注册到事件循环（Event Loop）中**排队执行。
- **工作原理**：Task 负责推动协程向前走。当协程遇到 `await` 暂停时，Task 会挂起；当 `await` 的 I/O 完成时，Task 会重新激活协程继续往下走。当协程执行完毕并 `return` 时，Task 会将这个返回值存入自己内部（因为它是一个 Future），并将状态改为 `DONE`。

----

### future,任务和协程之间的关系。

任务可以被认为是一个协程和一个future的组合。

任务的父类是future。

任务和协程的共同点是 awaitable抽象基类。
